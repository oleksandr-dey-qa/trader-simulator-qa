# 📈 Trader (Spribe) — QA & Player Checklist

Цей репозиторій містить практичний чекліст для безпечної гри та тестування стратегій у симуляторі торгівлі фінансовими активами **Trader** від відомого провайдера **Spribe**.

Проєкт створено в навчальних цілях для структурування процесу тестування логіки гравця, мінімізації ризиків «зливу» банку та контролю емоційного стану під час гри (тильт-менеджмент).

## 🗂 Структура репозиторію

*   [CHECKLIST.md](./CHECKLIST.md) — безпосередньо сам чекліст у форматі Markdown з інтерактивними чекбоксами.
*   [trader_spribe_checklist.csv](./trader_spribe_checklist.csv) *(опціонально)* — версія чекліста у форматі CSV для імпорту в TMS (наприклад, TestRail) або відкриття в Excel/Google Sheets.

## 🚀 Як користуватися чеклістом на GitHub

Ви можете використовувати цей чекліст кількома способами:

1.  **Прямо в GitHub Issues:**
    Створіть новий Issue у вашому репозиторії, скопіюйте туди вміст файлу `CHECKLIST.md` та відмічайте виконані пункти галочками прямо під час ігрової сесії. GitHub підтримує інтерактивні чекбокси!
2.  **Для імпорту в TestRail:**
    Завантажте файл `.csv` і замапте стовпці:
    *   *Категорія* $\rightarrow$ `Section`
    *   *Пункт чекліста* $\rightarrow$ `Title`
    *   *Опис / Ціль* $\rightarrow$ `Preconditions` або `Description`

Гра працює на базі генератора випадкових чисел (Provably Fair). Ніякі стратегії не гарантують 100% виграшу. Цей чекліст створений виключно для самоконтролю, менеджменту ризиків та тестування ігрової дисципліни.
